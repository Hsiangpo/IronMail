# IronMail 邮件模板说明

邮件模板用于统一管理邮件主题和邮件正文。发送前程序会先选择收件人名单，再选择邮件模板。

默认模板会演示 `示例.xlsx` 里的三个变量：

```text
邮件主题：关于 {{网页}} 的沟通

邮件正文：
{{法人}}您好，

我正在整理贵司公开信息，当前记录的网站是 {{网页}}，联系邮箱是 {{邮箱}}。

这里可以继续填写正式邮件正文。
                                          {}
```

这表示对象表不再放邮件内容，只放收件人和客户信息；邮件内容统一从模板里读取。

## 目录规则

- 收件人名单放在 `Mails/收件人名单/`
- 邮件模板放在 `Mails/邮件模板/`
- 模板文件必须使用 `.md` 后缀
- `README.md` 是说明文件，不会出现在模板选择列表里

## 收件人名单字段

必填字段：

```text
邮箱
```

`示例.xlsx` 当前字段：

```text
网页
法人
邮箱
```

其中：

```text
邮箱      = 发给谁
网页      = 客户网站，可作为变量
法人      = 客户姓名或负责人，可作为变量
```

以后新名单也可以增加这些可选字段：

```text
公司名
联系人
法人
备注
```

旧表仍然兼容。旧表里如果已经有 `邮件主题` 和 `邮件正文`，发送时可以选择不使用模板，继续按表格里的主题和正文发送。新表建议只保留对象信息，把邮件内容放到 `.md` 模板。

## 模板格式

模板第一行必须写邮件主题：

```text
邮件主题：关于 {{网页}} 的沟通
```

正文从 `邮件正文：` 后面开始：

```text
邮件正文：
{{法人}}您好，

我正在整理贵司公开信息，当前记录的网站是 {{网页}}，联系邮箱是 {{邮箱}}。
```

完整示例：

```text
邮件主题：关于 {{网页}} 的沟通

邮件正文：
{{法人}}您好，

我正在整理贵司公开信息，当前记录的网站是 {{网页}}，联系邮箱是 {{邮箱}}。
```

## 变量规则

变量写法固定为：

```text
{{字段名}}
```

字段名必须和收件人名单的表头完全一致。例如对象表有一列叫 `网页`，模板里就可以写：

```text
{{网页}}
```

发送时，程序会按每一行替换变量。

示例表格：

```text
网页: kf-ingenieure.de
法人: 马萨基
邮箱: buyer@example.com
```

模板：

```text
邮件主题：关于 {{网页}} 的沟通

邮件正文：
{{法人}}您好，我正在整理贵司公开信息，当前记录的网站是 {{网页}}，联系邮箱是 {{邮箱}}。
```

实际发送：

```text
邮件主题：关于 kf-ingenieure.de 的沟通

马萨基您好，我正在整理贵司公开信息，当前记录的网站是 kf-ingenieure.de，联系邮箱是 buyer@example.com。
```

也可以写成更像新话术的模板：

```text
邮件主题：关于 {{网页}} 的合作沟通

邮件正文：
{{法人}}您好，

我看到贵司网站是 {{网页}}，想和您简单沟通一下合作机会。
```

## 检查规则

- 模板里使用了 `{{网页}}`，对象表必须有 `网页` 这一列
- 模板里使用了 `{{法人}}`，对象表必须有 `法人` 这一列
- 模板里使用了 `{{邮箱}}`，对象表必须有 `邮箱` 这一列
- 如果某一行的变量值为空，会替换成空白
- 模板第一行没有 `邮件主题：`，程序会拦截
- 模板正文为空，程序会拦截
- 对象表没有 `邮箱`，程序会拦截
